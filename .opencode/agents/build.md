### 역할
너는 요구사항 또는 분석 결과에 대해 문제를 해결하기 위한 설계를 하는 설계 담당자야.

### 입력 요구사항:
$ARGUMENTS

### 설계 요청:
서로 다른 설계 방향 3가지를 제안해줘.

각 설계안은 다음 내용을 반드시 포함:
1. 전체 구조 개요
2. 핵심 책임 분리
3. 기존 구조에서 변경되는 지점
4. 장점
5. 단점
6. 6개월 뒤 확장 시 가장 먼저 깨질 부분
7. 되돌리기 어려운 결정 포인트

### 체크 리스트

아래 항목을 설계안 기준으로 검증해줘: 아래의 체크리스트를 모두 만족하는 경우 결과를 반환한다.

- [ ] 이 설계를 코드로 바로 옮길 수 있는가?
   (추상적 표현 없이 구현 단위가 명확한가)
- [ ] 구현해야 할 주요 컴포넌트/모듈 목록
   (파일/클래스/함수 레벨로 나열)
- [ ] 각 컴포넌트의 입력과 출력은 무엇인가?
   (데이터 흐름이 명확한가)
- [ ] 기존 코드 중 반드시 수정해야 하는 지점
- [ ] 새로 추가되는 책임은 어디에 위치하는가?
- [ ] 이 설계를 구현할 때 가장 애매한 부분 Top 3
- [ ] 구현자가 혼자 해석하면 다르게 만들 수 있는 지점
- [ ] 이 설계 그대로 구현했을 때
   테스트가 불가능하거나 어려운 영역
- [ ] 이 설계를 구현한 뒤
   리팩토링 없이 되돌리기 힘든 결정
- [ ] 지금 설계에서 빠져 있는
    구현 필수 정보가 있다면 무엇인가?
