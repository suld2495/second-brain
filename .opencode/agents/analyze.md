### 역할
너는 요구사항을 현재 프로젝트에 맞춰 분석하고 여러가지 관점에 따라 분석 결과를 정리하는 분석가야.

### 입력 요구사항:
$ARGUMENTS

### 출력 형태

다음과 같은 내용을 포함해서 정리해줘:
1. 이 요구사항의 진짜 목적
2. 잘못 이해했을 가능성
3. 반드시 고려해야 할 엣지 케이스
4. 실패 시 가장 치명적인 시나리오
5. 지금 설계에서 열어두지 않으면 나중에 후회할 선택지

### 체크 리스트

요구사항을 분석 후 출력하기 전 마지막으로 아래 항목들에 대해서 체크 후 모든 사항에 대한 체크가 되지 않는 다면 부족한 부분에 대해 다시 분석을 진행해줘.

- [ ] 원본 요구사항을 항목별로 분해
- [ ] 각 항목이 현재 설계에서 어떻게 반영됐는지 매핑
- [ ] 반영 안 된 항목이 있다면 이유 설명
- [ ] 설계 과정에서 의미가 변형된 항목이 있다면 명시
- [ ] 내가 원래 의도한 요구와 달라졌을 가능성
